# FINDINGS 2026-06-20 -- C-native calibration bench for the single-field Wirtinger judge

## What was tested
`cnative_bench.py` stress-tests `judge_v2.certify_1field` against an INDEPENDENT
exact Wirtinger reference oracle (separate from the judge). The oracle assigns a
5-class ground truth by construction (HOL / ANTI / MIXED / MODULE_TRAPPED /
REAL_TRAPPED), projects it onto the judge's 3 single-field labels
(holomorphic / anti-holomorphic / real-trapped), and asserts judge == expected.
Corpus: 23 hand-checked gold controls + 60 random expressions per class
(typed grammar over {+, -, *, exp, log, 1/.}, leaves {z, zbar, const}, depth 2).

- REAL_TRAPPED fully isolated (74/74): the 2026-06-18 full_conj fix holds on 74
  random real-valued forms, not only the 4 original calibration cases.

## Known limitation (measured, not fixed)
The single-field judge has no MODULE_TRAPPED or MIXED label: both collapse into
"anti-holomorphic". It does not distinguish a reducible module (z^a zbar^b =
z^(a-b)|z|^(2b)) from a genuine anti/mixed field. This is the next gap to close
before trusting the judge to rule a physical candidate as genuine-anti vs
module-trap.
