# Findings — 2026-05-22

Research session on the generative completeness of the EML operator family
{eml, eml★, eml⁰} and the boundary of what they can reconstruct.

All numerical results below were produced by execution on Anthony's Linux
machine (PySR) and/or by the exact SymPy judge. The arbiter is execution +
the exact Wirtinger judge, never a PySR marker or a reasoning argument.

## 1. Operator family classification — [ESTABLISHED]

**Test.** Enumerate all operators of the form `f(a) - g(b)` with
f, g in {id, exp, log, sin, cos} and a, b in {z, conj(z)}; classify each by
exact Wirtinger derivatives (df/dz, df/dzbar), z and zbar treated as
independent symbols.

**Command.** `python3 classify_eml_family.py`

**Raw result.** 100 operators: 20 holomorphic, 20 anti-holomorphic (pure),
50 hybrid, 10 constant (1 distinct). Named operators land correctly:
- eml = exp(z) - log(z)            -> holomorphic
- eml★ = exp(z) - log(conj z)      -> hybrid (canonical form A)
- exp(conj z) - log(conj z)        -> anti-holomorphic (= conj(eml))
- exp(conj z) - log(z)             -> hybrid (unnamed "mirror" of eml★)

**Status.** [ESTABLISHED] — exact symbolic classification, reproducible.

## 3. New base functions are reachable — [DERIVATION]

**Test (Phase 2, symbolic sieve).** Do sqrt, powers, 1/z, arctan, arcsin,
arccos, tan, sinh reduce to exp/log (hence reachable from {eml, eml★, Re})?

**Command.** `python3 sieve_phase2.py`

**Raw result.** 10/10 reachable, each with an explicit exp/log form, e.g.:
- sqrt(z)   = exp(log(z)/2)
- z^pi      = exp(pi*log(z))
- 1/z       = exp(-log(z))
- arctan(z) = (i/2)[log(1-iz) - log(1+iz)]

**Status.** [DERIVATION] — the explicit exp/log forms are proofs for each
case; the closure claim is a sound argument backed by SymPy rewriting, not a
single machine-checked theorem.

## 4. The infinity wall: Gamma is NOT reconstructible — [ESTABLISHED]

**Test.** Give PySR the FULL toolbox {eml, eml★, eml⁰, +,-,*,/, exp, log,
sin, cos, my_real, my_conj} with a LARGE budget, and ask it to reconstruct
the Gamma function. Compare to an elementary control (sin(z)*exp(z)) on the
SAME budget.

**Command.** `python3 test_gamma_full.py`

**Raw result (MSE).**
- control sin(z)*exp(z) (elementary) : 7.64e-30  (machine precision)
- Gamma(z)                           : 1.97e-03  (plateau, far above)
- Gamma is ~2.6e+26x worse than the elementary control.

**Interpretation.** Same budget, same toolbox: the control reaches machine
precision, Gamma plateaus at 1e-3. This is the numerical signature of
hypertranscendence (Hoelder 1887: Gamma satisfies no algebraic differential
equation). A finite operator system cannot reconstruct Gamma because Gamma
requires an infinite process (Weierstrass product / infinite sum); no finite
composition of operators encodes infinity. eml⁰ does not help: it unlocks the
PHASE/argument axis (a finite, geometric limitation), which is orthogonal to
the infinite-process axis of Gamma.

**Status.** [ESTABLISHED] — by execution. The control passing rules out
"insufficient budget" as the cause.

## 5. Two distinct walls — [DERIVATION]

The EML framework has two orthogonal limitations:
1. **Phase / monodromy axis** (finite, geometric): the argument Arg(z), the
   winding number. eml⁰ crosses part of this (isolates Arg). A finite 4th
   operator could in principle live here — OPEN, to be tested
   (test_phase_operators.py: bounded phase should reconstruct; unbounded
   winding may resist due to multivaluedness).
2. **Infinite-process axis** (transcendence): Gamma, zeta. NO finite operator
   crosses this (wall #4 above). Definitive.

Inversion / Mobius transforms (z -> 1/z, the 0<->inf bridge of the Riemann
sphere) are rational functions, already inside {eml}: holomorphic, reachable.
The "outside" Anthony intuits on the sphere is a DOMAIN question (monodromy),
not a new generative OPERATOR.

**Status.** [DERIVATION] — sound, partly confirmed by execution (1,2,4),
the winding/monodromy frontier (axis 1) not yet tested by execution.

## Conclusion (current)

[ESTABLISHED] {eml, eml★} reconstructs all elementary functions tested
(elementary holomorphic + anti-holomorphic), at machine precision (Phase 1)
or with explicit exp/log forms (Phase 2). No new elementary operator exists.

[ESTABLISHED] The framework stops at hypertranscendental functions (Gamma):
shown numerically (wall #4). This is a boundary, not a missing brick.

[OPEN] The phase/monodromy axis (winding number) is the only place a finite
4th operator could exist; not yet tested by execution.
