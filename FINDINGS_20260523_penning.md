# FINDINGS — Penning-map calibration (eml / eml-star detector)

**Date:** 2026-05-23
**Status:** [ESTABLISHED] — executed on Anthony's machine AND certified by the
exact SymPy judge (`verify_exact.py`, Wirtinger derivative d/d(zbar)).
**Nature:** CALIBRATION of the detector, **NOT a physical discovery.**

---

## 1. What was tested

Recovery of the holomorphic class of four natively-complex maps of the
transverse plane of a Penning trap (rho = x + i*y), as a sanity check of the
PySR + judge pipeline. The maps are generated by `simulate_penning_maps.py`.

**The data are SYNTHETIC.** Each map is produced from a known closed form, so
the expected class is known in advance. This run validates the detector; it
does **not** discover any new physics. It is the analogue of the SPARC lesson:
a class injected on purpose, recovered, is calibration — not a result.

| Map | Closed form (injected) | Ground-truth class |
|-----|------------------------|--------------------|
| penning_holo          | f = e^{i*theta} * z                                | holomorphic |
| penning_anti          | f = e^{i*theta} * conj(z)                          | anti-holomorphic |
| penning_hybrid        | f = cosh(r)*z + e^{i*phi} sinh(r)*conj(z) (squeeze)| hybrid (anti) |
| penning_holo_shuffled | HOLO with Im(output) randomly permuted             | NEGATIVE CONTROL |

Two operator toolboxes (double validation, no `my_conj` in either, one
anti-holomorphic door each):
- Route A: {eml, emlstar}  (emlstar is the native anti-holomorphic operator)
- Route B: {eml, my_real}  (anti-holomorphy reached via 2*Re(z) - z = conj(z))

## 2. Exact commands

Generate data:
    python3 simulate_penning_maps.py

Run the detector (deterministic + serial, paper-v8 reproducibility config):
    python3 -u double_validation_v6.py --datasets penning_holo penning_anti penning_hybrid penning_holo_shuffled
    # -> double_validation_v6_result.json

Certify each best_equation with the EXACT JUDGE (the only authority):
    python3 verify_exact.py --formula "<best_equation from JSON>"

## 4. Negative control — the key result

`penning_holo_shuffled` (output imaginary part randomly permuted, structure
destroyed) does NOT yield an exploitable fit:
- MSE stays at ~1.3-1.4 on BOTH routes,
- vs ~1e-24 .. 1e-32 on the genuine datasets — 24+ orders of magnitude gap,
- PySR piles operators up to maxsize=20 (including emlstar) with NO loss
  improvement past complexity ~10.

Per the project rule **MSE >= 1e-3 invalidates the claim**, so the shuffled
verdict is invalid regardless of any marker. The detector does NOT fabricate
structure on noise. This is the anti-SPARC guarantee.

## 5. Marker is NOT the judge — demonstrated live

`double_validation_v6.py` reports a `verdict`/`marker` field computed by a
**substring test** (`detect_anti_holomorphic = marker in equation_str`,
line 177-178). On the shuffled control this marker prints
"ANTI-HOLOMORPHIC (marker found)" on pure noise. The script flags this with
" !" (disagrees with judge). Conclusion confirmed:

- the PySR marker is INDICATIVE only;
- the authority is (a) MSE < 1e-3 AND (b) the exact SymPy judge d/d(zbar);
- never certify on a marker.
